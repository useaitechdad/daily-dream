**Direction change — please read fully before acting.**

We tested v0.2 (hybrid SVG body + Nano Banana AI face) with our actual target user, my 13-year-old daughter. Her feedback, which after reflection I agree with:

- The all-SVG Miis (like Luna — brown bob, blue dress, purple halo) look like a coherent cast of characters
- The AI-face Miis (like Koji — anime portrait on SVG body) look like a head pasted onto a doll. Specifically: (a) the face has rendering detail and shading the SVG body doesn't have, (b) the pastel-background prompt bleeds through as a visible halo behind the head, (c) the line weight and style don't match the body's flat-color outlines
- For a game where we'll display 6-12 Miis at once on an apartment screen, visual consistency matters more than individual-face expressiveness

Decision: **drop the AI face system entirely in phase 1.** Replace it with an expanded procedural SVG face system. Nano Banana may return in phase 3 for dream sequences (where stylistic *mismatch* with the waking world is the point) — but not here.

Before writing any code, produce an **updated plan artifact** showing exactly what you'll delete, rename, and add. I'll review and approve before you touch the codebase. Keep the existing design-system aesthetic (the rounded, soft, kid-friendly feel) — only the face layer and surrounding API plumbing are changing.

## What to add — the new SVG face system

Update the Mii schema (`.agents/skills/mii-schema.md`) to bump `schemaVersion` to `2` and replace the `face` object with these `appearance.face.*` fields (nested under the existing `appearance` object):

```json
"appearance": {
  "skinTone": "...",
  "bodyShape": "...",
  "outfit": { ... },
  "accessory": "...",
  "face": {
    "hairStyle": "bob | long | pigtails | spiky | curly | buzz | ponytail | afro",
    "hairColor": "#RRGGBB",
    "eyeShape": "round | almond | sleepy | wide | angry | sparkle",
    "eyeColor": "#RRGGBB",
    "mouthShape": "smile | smirk | grin | pout | flat | open",
    "eyebrows": "none | arched | flat | angry",
    "blush": true | false,
    "freckles": true | false
  }
}
```

Schema validation (`shared/schema/miiSchema.js` — formerly in `shared/schema/` per the implementation plan):

- Bump `SCHEMA_VERSION` to `2`
- Add a `migrateV1ToV2(mii)` function that takes a v1 record and returns a v2 record by:
  - Dropping the entire top-level `face` object
  - Adding sensible defaults under `appearance.face`: `hairStyle: "bob"`, `hairColor: "#2D1810"`, `eyeShape: "round"`, `eyeColor: "#2D1810"`, `mouthShape: "smile"`, `eyebrows: "arched"`, `blush: false`, `freckles: false`
- On load (`storage.js`), if a stored Mii is `schemaVersion: 1`, run it through `migrateV1ToV2` and rewrite it. Log a one-line info toast: "Updated {name} to new face system"
- Update all validators to accept v2 and reject v1 input coming from `importMiis`

Rendering (`apps/mii-designer/lib/miiRenderer.js`):

- Add one pure function per face-part type, each returning an SVG fragment:
  - `renderHair(style, color, headBox)` → 8 paths, one per style
  - `renderEyes(shape, color, headBox)` → 6 variants. Each renders two eyes, one per side. Eye spacing and vertical position are consistent — only the shape path changes.
  - `renderMouth(shape, headBox)` → 6 variants, always dark (inherit a stroke color from a constant)
  - `renderEyebrows(style, color, headBox)` → 4 variants (none renders nothing); color is derived from hairColor, darkened ~15%
  - `renderBlush(enabled, headBox)` → two soft pink circles on cheeks, or nothing
  - `renderFreckles(enabled, skinTone, headBox)` → a scatter of 6-8 small dots in a darkened skin tone, deterministic (same positions every render)
- The head remains the existing skin-tone ellipse. Face parts render on top of it in this z-order (bottom to top): blush → freckles → eyebrows → eyes → mouth → hair. Hair last so it can overlap the forehead.
- Keep the overall look harmonious with Luna — flat colors, clean rounded shapes, no shading, no gradients

Controls panel (`apps/mii-designer/index.html`, handlers in `main.js`, styles in `style.css`):

Replace the old "Face" section with a new collapsible section that contains:

- Hair style: dropdown OR a row of small clickable SVG thumbnails (thumbnails are nicer for a kid — do thumbnails if feasible without heavy lift, otherwise dropdown is fine)
- Hair color: color picker
- Eye shape: dropdown or thumbnail row
- Eye color: color picker
- Mouth shape: dropdown or thumbnail row
- Eyebrows: dropdown (4 options, "none" included)
- Blush: toggle
- Freckles: toggle
- **Big "🎲 Randomize face" button at the top of the section** — picks random values across all face parameters. This is the most important button in the app; make it prominent.

Default values for a new Mii and for the Example Mii (so the default Example matches Luna's look):

- `hairStyle: "bob"`
- `hairColor: "#3B2416"` (the brown from Luna's hair)
- `eyeShape: "round"`
- `eyeColor: "#2D1810"` (dark brown)
- `mouthShape: "smile"`
- `eyebrows: "arched"`
- `blush: true`
- `freckles: false`
